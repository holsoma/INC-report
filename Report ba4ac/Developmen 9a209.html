Folder Structure

Dev Environment SETUP

  1. Clone the project repository from GitHub 
git clone https://github.com/PaulineChew/INC-latest.git
  1. Set up a virtual environment and activate it
python -m venv venv
source venv/Scripts/activate
  1. Install all required libraries and packages
cd haystack
pip install --upgrade pip
pip install -e .[all] ## or 'all-gpu' for the GPU-enabled dependencies

If you cannot upgrade pip to version 21.3 or higher, you will need to replace:

  • [all] with [sql,only-faiss,only milvus1,weaviate,graphdb,crawler,preprocessing,ocr,onnx,ray,dev]
  • [all-gpu] with [sql,only-faiss-gpu,only-milvus1,weaviate,graphdb,crawler,preprocessing,ocr,onnx-gpu,ray,dev]

  1. Install REST API and Streamlit UI (from root directory of Haystack repository)
pip install rest_api/
pip install ui/

ElasticSearch

Install ElasticSearch locally, using a method of your choosing

https://www.elastic.co/guide/en/elasticsearch/reference/current/install-elasticsearch.html

You can also install the ElasticSearch MSI package (older version)

https://www.elastic.co/guide/en/elasticsearch/reference/6.8/windows.html

Starting ElasticSearch on port 9200 (using .bat file)

cd elasticsearch
./bin/elasticsearch

Starting ElasticSearch on port 9200 (as a service)

If installation was done with the default settings, ElasticSearch should run if you restart your machine.

REST API

Start the REST API (in a fresh Git Bash terminal)

cd haystack
uvicorn rest_api.application:app

Streamlit UI

Start the UI 

cd haystack
streamlit run ui/hydra_app.py

Your application should be running now. Enjoy!

Troubleshooting for HAYSTACK

ES TransportError

TransportError(429, 'cluster_block_exception', 
'index [document] blocked by: [TOO_MANY_REQUESTS/12/disk usage exceeded flood-stage watermark, index has read-only-allow-delete block];

Solution

curl -XPUT -H "Content-Type: application/json" http://localhost:9200/_cluster/settings -d '{ "transient": { "cluster.routing.allocation.disk.threshold_enabled": false } }'
curl -XPUT -H "Content-Type: application/json" http://localhost:9200/_all/_settings -d '{"index.blocks.read_only_allow_delete": null}'

REST_API ImportError

ModuleNotFoundError: No module named 'rest_api'

Solution(s)

cd rest_api
uvicorn application:app
cd haystack
uvicorn rest_api.application:app
cd haystack
gunicorn rest_api.application:app

ElasticSearch RequestError

RequestError(400, "search_phase_execution_exception: Likely some of your stored documents don't have embeddings. Run the document store's update_embeddings() method.", 'No documents with emeddings.')

Solution

from haystack.document_stores.elasticsearch import ElasticsearchDocumentStore
from haystack.retriever.dense import DensePassageRetriever

document_store = ElasticsearchDocumentStore(
    host="localhost",
    username='',password='')

retriever = DensePassageRetriever(
    document_store=document_store,
    query_embedding_model="facebook/dpr-question_encoder-single-nq-base",
    passage_embedding_model="facebook/dpr-ctx_encoder-single-nq-base"
)
document_store.update_embeddings(retriever)

Failed loading pipeline component

Failed loading pipeline component 'TextFileConverter': __init__() got an unexpected keyword argument 'split_paragraphs'

Solution

Go back and look at the which pipeline file you used under rest_api/config.py.

Check either the pipelines.yaml or pipelines_dpr.yaml file to see if you defined any components and their parameters incorrectly.

Check for syntax errors as well.

Guide to using the Golden Retriever

Installing Prerequisites

1) Download the file or clone it (https://github.com/aimakerspace/goldenretriever)

2) Check if you had docker install by running 

docker -v && docker-compose -v

3) Set up a virtual environment by running 

virtualenv env 
#or
python -m venv env

4) To open and use the virtual environment run 

source env/Scripts/activate

5) Install required packages by running (inside the virtual environment!)

 pip install -r requirements.txt

6) Create a .env file in the project root folder to manage Minio access keys. Example:

MINIO_URL=localhost:9001

ACCESS_KEY=minio # change this to a proper access key!

SECRET_KEY=minio123 # change to to a proper secret key!

Change the docker-compose.yml file, change the access key and secret key to your own settings.

At minio_handler.py , remove all the ResponseError, BucketAlreadyOwnedByYou, BucketAlreadyExists ¬†inside the file and change it all to just 1 error called InvalidResponseError

Building the containers

7) Build and start up the multi-container application

docker-compose -f docker-compose.yml build

Followed by

 docker-compose -f docker-compose.yml up

The app is supposed to exit whereas minio and elk is suppose to be runnning

CSV FORMATTING

Add -- in front of url, csv file and index_name at create_doc_index.py, at parser.add_arguements

CSV columns have to follow this sequence

ans_id

ans_str

query_id

query_str

Upload your datasets by running 

python -m src.elasticsearch.create_doc_index --url localhost --csv_file data/(yourcsvfilename).csv --index_name (yourownindexname)
  • url: elasticsearch url
  • csv_file: datasets (csv file with QA pairs)
  • index_name: name of index to create

Ammending setup configuration in app/api/config.py

  • QA_INDEX: index name that you created (e.g: test, (yourownindexname))
  • INDEX_BUCKET = 'test-index'
  • INDEX_PICKLE = 'test.pkl'
  • INDEX_FILE = 'test.idx'
  • INDEX_PREFIX = 'test'

10) Amend setup configuration in app/api/config.py.

11) To run the finetune scripts:

python -m src.finetune.main 
#you can amend src/finetune/config.py to insert preferred finetuning parameters

12) To encode your documents:

python -m src.dvc_pipeline_scripts.index_encode --data ./data/test.csv --output_folder model_artefacts --index_prefix test --gr_model USEEncoder --savedir ./test_finetune/model_test/USE/best/0/
  • -data: path to file with raw responses
  • -output_folder: path to save index
  • -index_prefix: prefix to add to -data.pkl of index
  • -gr_model: name of gr model
  • -savedir: path to fine-tuned encoder weights
  • -index_name the index name define earlier when uploading datasets(e.g:test)

Running the backend

13) To start up¬† backend app run 

uvicorn app.api.main_es:app

Running the frontend

14) Start the front end using: 

streamlit run app/streamlit/main.py -- --url http://localhost:8000
